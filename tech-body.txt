**Problem.** No mechanism exists to determine whether a commit's file changes are entirely within test paths or include non-test (implementation) changes. This classification is required before any Red Gate logic can distinguish "tests-only commits" from "implementation commits." Different sw2m repos use different test path conventions, so hardcoding paths is not viable.

**Done when.** Given a commit SHA and a PR context, the classifier returns a boolean: true if all changed files match test-path patterns, false otherwise. Test-path patterns are configurable per-repo (e.g., via a GitHub variable) with a sensible default when unconfigured. The classifier is usable by other CI jobs as a reusable step or script.
